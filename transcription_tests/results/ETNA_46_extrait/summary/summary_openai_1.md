## Sujets abordés

| Thème | Points clés |
|-------|-------------|
| **Réduction du budget DLE** | - Confirmation attendue des montants exacts. <br>- Impact limité mais possible “coup de rabot” sur certains projets. |
| **Suivi des jalons** | - Besoin de dates de livraison plus précises (pas seulement “avant/après l’été”). <br>- Audran souligne l’importance d’une planification fine pour rassurer la direction et les parties externes. |
| **Relation avec les prestataires** | - Manque de visibilité sur les road‑maps (Collabora, BVB, Nextcloud). <br>- Décalages fréquents entre les engagements contractuels et les livraisons réelles. <br>- Nécessité d’obtenir des engagements plus réalistes et de suivre les évolutions (ex. version 3.1, 24.0.4). |

| **Gestion des développements financés** | - Possibilité d’être plus exigeant sur les livrables financés (ex. demander un patch si la version majeure n’est pas disponible). |
| **Comparaison avec les outils internes (Tribu)** | - Les outils développés en interne offrent plus de contrôle sur les dates que les solutions tierces. |

## Décisions prises
1. **Améliorer la précision des jalons**
   - Audran devra fournir, d’ici la prochaine réunion, un tableau récapitulatif des dates prévues (ou fourchettes) pour chaque livrable majeur (septembre / octobre).

2. **Renforcer le suivi des prestataires**
   - Instaurer un **processus de suivi mensuel** des engagements (dates, livrables, road‑map) avec Collabora, BVB et Nextcloud.
   - Documenter les écarts entre les engagements contractuels et les livraisons réelles.

3. **Gestion de la régression Nextcloud 31**
   - Benoît présentera un rapport détaillé (impact fonctionnel, plan de mitigation) lors du prochain point « nuage ».

4. **Exigence sur les développements financés**
   - Les équipes projet devront, pour chaque évolution financée, demander un **engagement de date** ou, à défaut, un **patch** sur la version précédente.
